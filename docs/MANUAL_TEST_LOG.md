# Журнал ручного тестирования нефункциональных требований

## Применённые виды ручного тестирования

1. **Smoke-testing** — проверка факта запуска и базовой работоспособности (UX-01, UX-02).
2. **Sanity testing** — быстрая проверка ключевой функциональности после сборки (UX-09, REL-03).
3. **Usability testing** — оценка удобства интерфейса (UX-03..UX-13).
4. **Boundary testing** — граничные значения rows = 2 и rows = 100 (REL-03, REL-04).
5. **Stress testing** — текст 50 000 символов (SEC-07).
6. **Endurance testing** — длительная работа окна 30 минут (REL-06).
7. **Accessibility testing** — проверка работы только с клавиатурой (UX-13).
8. **Compatibility testing** — разные DPI и среды запуска (COMP-01..COMP-04).
9. **Localization testing** — RU/EN тексты, смешанный ввод (LOC-01..LOC-05).
10. **Negative testing** — null, пустые строки, инъекции (SEC-01, SEC-02, SEC-05, SEC-06).
